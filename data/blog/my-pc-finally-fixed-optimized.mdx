Look at the **Hardware reserved** section, it taking 936 Mb of my RAM. While it is not that big, but in this economy where all computer components are ridiculously expensive, **0.9 Gb space** is still a space that I don't want to waste. I understand that hardware reserved is often for the integrated GPU. But since my system has a dedicated GPU, why leave it unused? So, I wanted to reclaim that space.

I searched online and found one post from [Reddit](https://www.reddit.com/r/ASRock/comments/1ij6840/i_fixed_my_hardware_reserved_ram_problem/?utm_source=share&utm_medium=web3x&utm_name=web3xcss&utm_term=1&utm_content=share_button). I could not find the specific setting the OP mentioned. So, I turned to an LLM to guide me. Below are steps to disable iGPU and claim back a bit of the hardware reserved RAM space. The steps might be specific to my motherboard, the ASRock A620M-HDVM.2, but you can try to find similar settings in your BIOS if you have the same issue.

1. Boot into BIOS settings.
2. Go to the **Advanced** tab.
3. Enter the `AMD CBS` menu. *(CBS stands for **C**ommon **B**IOS **S**ettings)*
4. Enter the `NBIO Common Options` menu. *(NBIO stands for **N**orth**B**ridge **I**nput/**O**utput)*
5. Enter the `GFX Configuration` menu.
6. Find the `dGPU Only Mode` setting, and change it from `Auto` to `Enabled`.

<figure className="mx-auto w-full max-w-3xl">
  <img
    className="w-full h-auto"
    src="https://res.cloudinary.com/iqfareez-cloud/image/upload/v1776692169/website/ram-issue-blog-post/dGPU_only_mode_ASrock_ommtdu.jpg"
    alt="Turn on dGPU only mode in ASRock BIOS"
  />
  <figcaption className='text-center'>The settings originally set to Auto. Change it to Enabled</figcaption>
</figure>

Save the settings and exit BIOS. After booting into Windows, check your Task Manager again. The hardware reserved RAM space should be reduced, and I now have 15.6 Gb of usable RAM instead of 15.1 Gb. I admit that this is not a huge difference, but at least now the previously reserved RAM space is being useful to the system.

## Update BIOS fixes RAM issues

For the longest time since I built my first PC on 2024, I cannot configure my RAM to run at its advertised speed. I have a single 16 GB RAM Adata XPG Lander DDR5 that is advertised to run at 6000 MT/s. It have an EXPO profile that I can enable in the BIOS. However, when I activate the profile, my PC will randomly crashes or stopped responding while using it. I had to force shutdown it before starting it again.

I thought that perhaps I received *lemon-unit*[^1] RAM, I accepted the fate. I turn down the speed manually to 5400 MT/s, and my PC ran without any issue. Despite not running at its max speed, it's not the end of the world, and I can live with it. Because finally I can work and play games without random crashes.

## Change automatic SWAP drive settings

<Admonition type="note">
  In this section, I will call Windows Pagefile as SWAP, the one that Linux uses.
</Admonition>

When doing heavy work, such as compiling code with the emulator running, my PC will stutter. The UI will freeze including the mouse cursor. I have to wait for few seconds before it can respond again, but half the time, it never responded again and I had to force shutdown it. 

This happens somewhat rare, but when it happens, it is very annoying. One time I managed to open Task Manager while the stuttering is happening, I found out that my disk usage is at 100%. And who's using it? The **System**. Huh?

I looked up online, one Reddit post says it's normal because Windows might doing maintainance in the background, eg defragmenting the disk, or updates. But I don't think that's the case. It only happens when I'm doing heavy work, even after I restarted my PC!

Again, I turned to an LLM to ask for help. It it gives me some possible causes, the number 3 stands out to me.

Now, how do I prevent Windows from using my HDD as a SWAP? You can follow this steps:

1. Search `View advanced system settings`. Open it.
2. Go to `Advanced` tab, and click `Settings` in the `Performance` section.
3. Go to `Advanced` tab, and click `Change` in the `Virtual memory` section.
4. Uncheck `Automatically manage paging file size for all drives`. This is the culprit that causing the issue. Windows is using the HDD. I think it looks the most free space available.

You need to select which drive you allow Windows to use as a SWAP. I select only my SSD drive.

After I restarted the PC. I recompiled my app. The compilation success and I didn't encounter any stuttering or freezing issue. I can say that the issue is finally fixed! 

So, here's what I think is originally happening. When I'm compiling a codebase, my RAM is under pressure, i.e. when a lot of data goes in and out. Windows will try to use the SWAP drive to offload some of the data from RAM to the disk.

Unfortunately, Windows is 'smart' to choose my HDD as the SWAP drive, for whatever reason. Due to HDD is slower compared to SSD, it will cause high disk usage as the IO operation goes which slows the whole computer down.
